1.1 理解对象(简单介绍)

1.2单例模式

  • 单例模式的核心：确保只有一个实例，并提供全局访问。
  • 单例模式的作用：把描述同一件事物的属性和方法放在同一堆内存中，起到分组的作用，防止冲突；这样不同事物间即使属性名一样也不会发生冲突。
  • 在单例模式中把对象名叫做命名空间。
  • 单例模式的优点：
    1. 实现分组
    2. 防止变量名冲突
    3. 模块化开发：对于一个较大的项目，需要多人协同开发，根据项目需求划分成几个功能板块，每个人负责一部分，同时开发，最后把每个人的代码进行合并

2 工厂模式

单例模式的缺点
单例模式虽然解决了分组的问题，但是在很多时候并不能满足我们的需求。如1.1中的例子可以看出，person1，person2有很多一样的地方，但是却写了两遍，造成代码的耦合。

工厂模式的出现就很好的解决了这个问题

工厂模式把实现同一件事情的代码放到一个函数中，如果以后想实现这个功能，就不需要重新编写代码了，只需执行当前函数就可以了，避免手动去创建对象，减少页面中的冗余代码，提高代码的重复利用率。

问题点：虽然工厂模式解决了代码重复问题，但是无法解决实例的识别问题，因为工厂模式产出的对象都是Object类型的。因此我们需要引入构造函数来解决这个问题。

3 构造函数

3.1 构造函数的目的

构造函数模式的目的就是为了创建自定义类，并且创建这个类的实例，构造函数模式拥有了类和实例的概念，并且实例和实例之间是相互独立的，彼此互不干扰。

3.2 构造函数和普通函数的区别

  • 书写形式上：
  1. 从形式上看，一个函数被作为构造函数还是普通函数执行的唯一区别，看是否使用了new运算符。
  2. 执行的时候，构造函数new Person();而createPerson()只是普通函数执行。
    通过new执行后，Person就是一个类了。
  3. 约定俗成的规范是首字母大写:new Person();newCreatePerson();
  • 函数代码执行的时：
    • 相同点：都是形成一个私有作用域，然后形参赋值，然后预解析，代码从上至下执行(类和普通函数一样，类也具有普通函数的一面)；
    • 不同点：
    1. 构造函数在代码执行之前，不用手动创建对象，浏览器会默认创建一个对象数据类型的值(这个对象就是当前类的实例，以当前实例为主体，将属性名和属性值赋给实例,this代表当前实例)，浏览器会默认将创建的实例返回。
    2. this在构造函数中，是当前类的实例。
    3. 虽然各个实例p1,p2都是CreatePerson的实例，都拥有write方法，但是不同实例之间的方法是不一样的；实例和实例之间是单独的个体，所以私有属性之间不相等。

4.原型链模式

构造函数的问题：虽然构造函数解决了实例类型的问题，但是却无法解决方法被复用的问题：如每个实例都有一个write方法，而且方法是一样的，每次初始化实例的时候都会被重新生成，很占用内存，所以我们需要一个属性来挂载所有实例共有的方法和属性，而这正是原型对象。

4.1 原型模式三两句

  1. 每一个函数数据类型(普通函数、类)都有一个天生自带的属性:prototype(原型)，并且这个属性是一个对象数据类型的值。
  2. 在prototype上，浏览器自动给它增加了一个constructor(构造函数)属性，属性是当前函数或类本身。
  3. 每一个对象数据类型(普通对象，实例，prototype)也自带一个属性:_ proto _(两边各自两个下划线),属性值指向当前实例所属类的原型。

4.2 原型的作用

基于构造函数的原型模式，解决方法或属性共有的问题，把实例之间相同的属性和方法提取到CreatePerson.prototype上即可

4.3 原型链查找机制

原型链机制
我们都知道每个对象都有toString方法，可是我们在创建对象时并没有创建此函数，那它是从哪里来的呢？

  • 首先在对象的私有属性上进行查找，如果私有存在这个属性（方法），则获取的是私有的属性值（方法）；如果私有的没有这个属性（方法），则通过_ proto _找到所属类的原型，原型上定义的属性和方法都是当前实例的公有属性和方法，原型上存在的话，获取的是公有的属性值（方法），如果原型上也没有，则通过原型上的proto继续向上查找，一直找到Object的prototype为止。这样就够成了一个原型链：

5 继承

子类继承父类中的属性和方法

5.1 原型继承方式

  1. 子类B继承父类A的所有属性和方法(私有和公有的)的方式，A的实例既有私有属性方法，又有公有属性和方法，所以用A的实例作为B的原型地址
  2. 继承特点：把父类的私有的和公有的都继承到子类原型上，都是子类公有的
  3. 核心：原型继承并不是把父类的属性和方法克隆一份给子类，而是让父类和子类之间增加了原型链的链接，以后子类的实例想要用父类的方法，需要一级级的向上查找来使用
